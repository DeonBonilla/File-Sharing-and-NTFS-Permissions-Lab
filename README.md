# 📂 File Sharing and NTFS Permissions Lab

## 📋 Overview

This project documents the configuration of file sharing and NTFS permissions within a Windows Server environment. The lab was designed to simulate how organizations securely manage shared resources by controlling user access through NTFS permissions, share permissions, and Active Directory security groups.

Through this project, I gained hands-on experience implementing the principle of least privilege while ensuring users had the appropriate level of access to shared resources.

---

## 🛠️ Technologies Used

- Windows Server 2022
- Active Directory Domain Services (AD DS)
- NTFS Permissions
- Windows File Sharing
- Security Groups
- Windows 11
- Oracle VirtualBox

---

## 🎯 Objectives

- Create shared folders
- Configure NTFS permissions
- Configure Share permissions
- Assign access using Security Groups
- Test user access to shared resources
- Verify permission inheritance
- Implement least privilege access

---

## 🔧 Tasks Completed

### 📁 Shared Folder Configuration

Created and configured shared folders to simulate departmental file storage within an enterprise environment.

---

### 🔐 NTFS Permission Management

Configured NTFS permissions to control access based on user roles and security groups.

---

### 👥 Share Permission Configuration

Applied share permissions and verified that users could only access resources they were authorized to use.

---

### ✅ Access Validation

Tested user access using multiple domain accounts to verify that permission settings functioned correctly.

---

### 🛡️ Least Privilege Implementation

Configured permissions using the principle of least privilege to reduce unnecessary access to sensitive resources.
